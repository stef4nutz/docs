Impose restrictions on members with administrative permissions to prevent destructive behavior.

<Tip>
  We recommend using our dashboard for a easier use of the bot.
</Tip>

# Why use an antinuke system?

It's important to have an antinuke in place in order to prevent members with administrative permissions from adding random bots or changing your server structure. This measure gives you a freedom of mind knowing your server it's secure with our bot.

# How does the antinuke work?

The antinuke will set a limited number of actions for all the administrative members, which will prevent destructive behavior if the limit has been exceeded. The member that exceeded gets the punishment set & a message will be sent to the owner.

# Configuring the antinuke

Only the server owner can configure antinuke, however, you can allow other members you trust to also have access to these commands. You can do this by using `antinuke admin` or our dashboard.

# Setting up punishments for the antinuke

You can enable punishemnts for the antinuke by using `antinuke [module] [enable/disable] [ban/kick/strip] [limit]` but we heavily recommend using our dashboard on this matter since it's easier to setup.
